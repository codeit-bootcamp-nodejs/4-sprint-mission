
## 요구사항

### 기본
- [x] User 스키마 작성 
 - id, email, nickname, image, password, createdAt, updatedAt 필드를 가집니다
- [x] 회원가입 API 생성 
 - email, nickname, password 를 입력하여 회원가입을 진행
 - Password 해싱하여 저장
- [x] 토큰 기반 인증 
 - Access Token을 발급하는 기능을 구현
- [x] 상품 기능 인가 
 - 로그인한 유저만 상품 등록, 상품을 등록한 유저만 해당 상품 정보 수정 삭제
- [x] 게시글 기능 인가 
-  로그인한 유저만 게시글 등록, 해당 유저만 해당 게시글 수정 삭제
- [x] 댓글 기능 인가 
 - 로그인한 유저만 상품,게시글에 댓글 등록 및 해당 댓글 수정 삭제 
- [x] 유저 정보 
 -자신의 정보 조회, 수정, 비밀번호 변경, 등록한 상품 목록 조회 기능
 - 비밀번호 리스폰스로 노출x )

### 심화
- [x] Refresh Token 갱신하는 기능 구현
- [x] 좋아요 기능
 - 로그인한 유저는 상품에 좋아요, 좋아요 취소
 - 로그인한 유저는 게시글에 좋아요, 좋아요 취소
 - 상품 및 게시글 조회할 때 좋아요를 누른 항목인지 isLiked로 구현
 - 유저가 좋아요 표시한 상품의 목록 조회하는 기능
